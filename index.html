HTML Documentation

HTML (HyperText Markup Language)

HTML is the standard markup language for creating web pages and web applications. It is used to structure and format the content on the web, including text, images, and videos, and it allows web browsers to interpret and display the content.

HTML is written using tags, which are enclosed in angle brackets. These tags tell the browser how to format the content and where to display it on the page. There are a wide variety of HTML tags, each with its own specific purpose and meaning. For example, the <p> tag is used to define a paragraph, the <img> tag is used to include an image, and the <a> tag is used to create a link.

Here is a simple example of an HTML document:

<html>
<head>
<title>My Web Page</title>
</head>
<body>
<h1>Welcome to my web page!</h1>
<p>This is a simple web page.</p>
<ul>
<li>Item 1</li>
<li>Item 2</li>
<li>Item 3</li>
</ul>
<p><a href="https://www.example.com">Visit my website</a> for more information.</p>
</body>
</html>

This HTML document has a <head> element, which contains metadata about the document, and a <head> element, which contains the actual content of the page. The <h1> tag is used to define a heading, the <p> tag is used to define a paragraph, and the <ul> and <li> tags are used to create an unordered list. The <a> tag is used to create a link to another web page.

HTML is a powerful and flexible language that is essential for creating and developing websites and web applications. It is the foundation of the modern web and is used in conjunction with other technologies, such as CSS and JavaScript, to create rich and interactive user experiences.

HTML Tags

HTML tags are used to mark up the content on a web page. They indicate the structure and meaning of the content, and help browsers to render the content correctly.

HTML tags are enclosed in angle brackets, and typically come in pairs: an opening tag and a closing tag. The closing tag has a forward slash before the tag name. For example:

<p>This is a paragraph.</p>

In this example, <p> is the opening tag and </p> is the closing tag. The content of the paragraph, "This is a paragraph," appears between the two tags.

There are many different HTML tags that can be used to mark up a web page. Some of the most common include:

  • <html> : This tag indicates the start of an HTML document.
  • <head> : This tag contains information about the document, such as the title and any external style sheets or scripts.
  • <title> : This tag specifies the title of the document, which is displayed in the browser's title bar or tab.
  • <body> : This tag indicates the start of the document's main content.
  • <h1> to <h6> : These tag defines the headings, with <h1> being the most important and <h6> beaing the least imaportant.
  • <p> : This tag defines a paragraph.
  • <br> : This tag inserts a line break.
  • <em> : This tag italicizes text.
  • <strong> : This tag boldface text.
  • <img> : This tag inserts an image.
  • <a> : This tag creates a hyperlink.

There are many other HTML tags that you can use to create more advance pages. You can find a complete list of HTML tags at the following URL: http://www.w3schools.com/tags/.

Empty Elements

Empty elements are HTML elements that have no content and close themselves in the opening tag. They are also known as void elements or singleton elements.

Empty elements do not have an end tag in HTML instead, they are written with a start tag, and then a slash before the closing angle bracket.

Some common empty elements include:

  • <br> : inserts a line break
  • <hr> : inserts a horizontal rule (a line that runs across the page)
  • <img> : inserts an image
  • <input> : creates an input field
  • <meta> : provides the metadata(additional information about the page that can be used by the browsers) about the document
  • <link> : links to external resources, such as stylesheets

Here's an example of an empty element:

<img src="image.jpg" alt="A description of the image">

Empty elements do not have a closing tag, so the element above would be considered "empty."

Doctype Declaration

In HTML, the doctype declaration is used to specify the type of the document. It is placed at the very beginning of an HTML document, before the HTML element, and consists of a single line of code. The doctype declaration is not an HTML element and does not require a closing tag.

The doctype declaration is important because it tells the web browser how to interpret the HTML code in the document. It specifies which version of HTML the document is written in, and this information is used to ensure that the browser renders the content correctly.

Here is an example of the doctype declaration for HTML5:

<!DOCTYPE html>

This doctype declaration is used for all HTML documents written in the HTML5 specification.

In older version of HTML, the doctype declaration was longer and more complex. For example, here is the doctype declaration for HTML4.01:

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

It is generally recommended to use the HTML5 doctype declaration, as it is the most current and widely supported version of HTML.

HTML Attributes

HTML attributes provide additional information about HTML elements. They are specified in the opening tag of an element and are used to configure or modify the element in some way.

Attributes are made up of a name and a value, separated by an equals sign. The value should always be enclosed in quotation marks.

For example, the href attribute is used to specify a link for an a element:

<a href="https://www.example.com">Click here</a>

Here, the attribute name is href and the attribute value is "https://www.example.com".

Other common attributes include class, which is used to apply CSS styles to an element, and id, which is used to uniquely identify an element on a page.

<div class="red-text" id="main-content">This text will be red and have an id of "main-content"</div>

Attributes can also be used to specify the dimensions of elements, such as the width and height of an image:

<img src="image.jpg" width="200" height="100">

There are many other attributes available in HTML, and the specific attributes that are available for a given element depend on the element type. You can find more information about the different attributes that are available in the HTML specification.

HTML Links

HTML links are used to connect web pages to one another. They allow you to click on a piece of text or an image and be taken to a different web page, or to a specific location within the current page.

Here's an example of an HTML link:

<a href="http://www.example.com">Click here to go to Example.com</a>

The a element is used to create a link, and the href attribute specifies the destination of the link. The text between the a tags is the text that will be displayed as the link.

Target attribute

You can also use the target attribute to control where the linked page should be opened. It can have one of the following values:

  • _self: Default, Opens the document in the same window/tab as it was clicked
  • _blank: Opens the document in a new window or tab
  • _parent: Opens the document in the parent frame
  • _top: Opens the document in the full body of the window

For example, to open the link in a new browser window or tab, you can use target="_blank"

<a href="http://www.example.com" target="_blank">Click here to go to Example.com in a new window</a>

Bookmarks

Links can also be used to jump to a specific location within the same web page. This is done by using a link with a href attribute that begins with a # symbol, followed by the ID of the element you want to link to.

Here's an example:

<a href="#section1">Jump to Section 1</a>
...
<h2 id="section1">Section 1</h2>

In this example, clicking on the link will take the user to the element with the ID "section1" on the same page. We can also jump to another page's bookmark (href="anotherpage.html#section2")

mailto

A "mailto:" link is a type of HTML link that allows a user to send an email to a specific email address by clicking on the link. When the user clicks on the link, their default email client (e.g., Microsoft Outlook, Gmail) will open a new email message with the "To:" field already filled in with the email address specified in the link.

Here is an example of a mailto: link:

<a href="mailto:info@example.com">Send an email to info@example.com</a>

When the user clicks on this link, their email client will open a new message with the "To:" field already filled in with "info@example.com". The user can then enter a subject and message and send the email as they normally would.

You can also specify other parameters in the mailto: link to pre-populate other fields in the email message, such as the subject, cc, and bcc fields. For example:

<a href="mailto:info@example.com?subject=Inquiry&cc=sales@example.com&bcc=marketing@example.com">Send an email to info@example.com</a>

In this example, when the user clicks on the link, their email client will open a new message with the "To:" field already filled in with "info@example.com", the "Subject:" field already filled in with "Inquiry", and the "Cc:" and "Bcc:" fields already filled in with "sales@example.com" and "marketing@example.com", respectively.

Title attribute in anchor tag

The title attribute in an anchor tag <a> specifies a text to be displayed as a tooltip when the mouse pointer hovers over the anchor element. The title attribute is commonly used to provide a brief description or additional information about the linked content.

Here is an example of using the title attribute in an anchor tag:

<a href="https://example.com" title="Learn more about Example">Example</a>

In this example, the text "Learn more about Example" will be displayed as a tooltip when the mouse pointer hovers over the anchor element.

It's important to note that the title attribute is not intended to provide additional information about the link's destination. Instead, it should be used to provide additional context or information about the link itself. If you want to provide information about the destination of the link, it's generally a better practice to include that information in the linked content itself.

Absolute and Relative URLs

An absolute URL is a complete web address that includes the protocol (such as "http" or "https"), the domain name, and the path to a specific file or resource.

A relative URL, on the other hand, is a web address that is relative to the current page. It specifies the path to a file or resource using only the necessary information, and does not include the protocol or domain name.

For example, consider a page located at "https://www.example.com/path/to/page.html". If this page includes a link to an image located at "https://www.example.com/path/to/image.jpg", the URL for the image would be an absolute URL. If the page includes a link to another page located at "../otherpage.html", the URL for the other page would be a relative URL.

Absolute URLs are useful when linking to resources that are not on the same server or domain as the current page. Relative URLs are useful when linking to resources that are on the same server or domain, as they allow you to specify the path to the resource relative to the current page, making it easier to move pages around within a site without having to update all of the URLs.

Images in HTML

To display an image in an HTML page, you can use the img element. Images are not technically inserted into a web page, images are linked to web pages, The img tag creates a holding space for the referenced image. The img element is an empty element, which means it has no content and does not require a closing tag.

Here is an example of how you can use the img element to display an image in an HTML page:

<img src="image.jpg" alt="A description of the image">

The src attribute specifies the URL of the image. The alt attribute provides an alternate text for the image, which can be displayed if the image cannot be displayed or if the user is using a screen reader.

You can also specify the width and height of the image using the width and height attributes, respectively. For example:

<img src="image.jpg" alt="A description of the image" width="200" height="100">

This will display the image with a width of 200 pixels and a height of 100 pixels.

You can also use the img element to display images that are stored on other websites by specifying the URL of the image in the src attribute. For example:

<img src="https://www.example.com/image.jpg" alt="A description of the image">

Note that you need to have permission to use an image from another website.

Favicon

A favicon (short for "favorite icon") is a small, iconic image that represents your website or brand. It is displayed in the browser tab, bookmark, and in the history of the browser.

To add a favicon to your HTML page, you can use the link element in the head of your HTML document. The link element should have a rel attribute with a value of "icon" or "shortcut icon", and a href attribute with the URL of the favicon image file. Here's an example:

<link rel="shortcut icon" href="/path/to/favicon.ico">

The favicon file should be a small image in .ico format. You can also use other image formats such as .png or .gif, but .ico is the most widely supported format.

It's a good idea to provide multiple sizes of the favicon image to support different devices and resolutions. You can do this by using the link element with different sizes attributes, like this:

<link rel="icon" type="image/png" sizes="16x16" href="/path/to/favicon-16x16.png">
<link rel="icon" type="image/png" sizes="32x32" href="/path/to/favicon-32x32.png">

You can also use the link element with a rel attribute of "apple-touch-icon" to specify a favicon for Apple devices:

<link rel="apple-touch-icon" href="/path/to/apple-touch-icon.png">

Note that the favicon may not be displayed immediately when you add it to your HTML page. The browser may need to load the page again or the user may need to refresh the page to see the favicon.

Tables

HTML tables allow you to present data in a grid format, making it easy for users to read and compare information. Let's get started with the basics.

  1. Creating a table

    2. To create a table in HTML, you will need to use the "table" element. Inside the table element, you can define rows using the "tr" element and cells using the "td" element. Here is an example of a basic table with two rows and two cells in each row:

    Sample Input
    <table>
    <tr>
    <td>Cell 1</td>
    <td>Cell 2</td>
    </tr>
    <tr>
    <td>Cell 3</td>
    <td>Cell 4</td>
    </tr>
    </table>
    Sample Output
    Cell 1 Cell 2
    Cell 3 Cell 4
  2. Adding a table header

    3. If you want to add a row at the top of your table with column headings, you can use the "th" element instead of "td". Here is an example of a table with a header row:

    Sample Input
    <table>
    <tr>
    <th>Column 1</th>
    <th>Column 2</th>
    </tr>
    <tr>
    <td>Cell 1</td>
    <td>Cell 2</td>
    </tr>
    <tr>
    <td>Cell 3</td>
    <td>Cell 4</td>
    </tr>
    </table>
         Sample output
    Column 1 Column 2
    Cell 1 Cell 2
    Cell 3 Cell 4
  3. Adding table borders

    4. By default, tables do not have borders. If you want to add borders to your table, you can use the "border" attribute on the "table" element. For example:

    Sample Input
    <table border="1">
    <tr>
    <th>Column 1</th>
    <th>Column 2</th>
    </tr>
    <tr>
    <td>Cell 1</td>
    <td>Cell 2</td>
    </tr>
    <tr>
    <td>Cell 3</td>
    <td>Cell 4</td>
    </tr>
    </table>
         Sample Output
    Column 1 Column 2
    Cell 1 Cell 2
    Cell 3 Cell 4
  4. Adding cell padding and spacing

    5. You can use the "cellpadding" and "cellspacing" attributes to add padding (space inside the cell) and spacing (space between cells) to your table. For example:

    Sample Input
    <table border="1" cellpadding="10" cellspacing="5">
    <tr>
    <th>Column 1</th>
    <th>Column 2</th>
    </tr>
    <tr>
    <td>Cell 1</td>
    <td>Cell 2</td>
    </tr>
    <tr>
    <td>Cell 3</td>
    <td>Cell 4</td>
    </tr>
    </table>
         Sample Output
    Column 1 Column 2
    Cell 1 Cell 2
    Cell 3 Cell 4
  5. Merging cells

    6. Sometimes you may want to merge cells in your table to create larger cells. You can do this using the "colspan" and "rowspan" attributes. For example, if you want to merge two cells into one cell, you can use the following code:

    Sample Input
    <table border="1">
    <tr>
    <td colspan="2">Merged Cell</td>
    </tr>
    <tr>
    <td>Cell 1</td>
    <td>Cell 2</td>
    </tr>
    </table>
         Sample Output
    Merged Cell
    Cell 1 Cell 2

    This will result in a table with one row and two columns, with the first column being a merged cell containing the text "Merged Cell" and spanning across both columns. The second row will contain two separate cells with the text "Cell 1" and "Cell 2".

    You can also use the rowspan attribute to merge cells vertically. For example:

    Sample Input
    <table border="1">
    <tr>
    <td rowspan="2">Merged Cell</td>
    <td>Cell 1</td>
    </tr>
    <tr>
    <td>Cell 2</td>
    </tr>
    </table>
         Sample Output
    Merged Cell Cell 1
    Cell 2
